📞 Cuộc gọi (Call Tracking)

POST /api/bookings/:id/calls Auth
Ghi nhận cuộc gọi + tự động chuyển trạng thái
📥 Request Body:{
  "outcome": "APPOINTMENT_BOOKED",  // * bắt buộc
  "notes": "Đã tư vấn xong",
  "next_call_at": "2026-03-25T10:00",
  "care_reminder_at": "2026-03-26T09:00",
  "appointment_at": "2026-03-25T14:00"   // bắt buộc nếu outcome = APPOINTMENT_BOOKED
}

Outcome hợp lệ:NO_ANSWER, WRONG_NUMBER, NOT_INTERESTED, CALL_BACK_LATER,
CARING, CONSULTED, APPOINTMENT_BOOKED, ARRIVED,
CLOSED_FIRST_REVENUE, LOST

🔄 Auto status transition:NO_ANSWER → CALLED        CALL_BACK_LATER → FOLLOW_UP
APPOINTMENT_BOOKED → APPOINTED   ARRIVED → ARRIVED
CLOSED_FIRST_REVENUE → WON       LOST → LOST

💡 Không bao giờ hạ cấp trạng thái. VD: APPOINTED → sẽ không bị chuyển về CALLED.
GET /api/bookings/:id/calls Auth
Lấy lịch sử cuộc gọi của 1 lead
📤 Response:{ "calls": [ { "call_number": 1, "outcome": "NO_ANSWER", "notes": "...", "call_time": "..." } ] }

1️⃣ Xác thực (Authentication)

Mọi API (trừ /api/bookings POST public và /api/login) đều cần gửi token trong header:

Authorization: Bearer <token_nhận_từ_login>

Bước 1: Gọi POST /api/login với username + password → nhận token

Bước 2: Mọi request sau đều gửi header Authorization: Bearer token

Bước 3: Token hết hạn sau 24h, server trả 401 → gọi login lại

fetch('https://hoanmy.emarketervietnam.vn/api/bookings', {

  body: JSON.stringify({

.catch(err => alert('Có lỗi xảy ra'));

6️⃣ Trạng thái Lead (Status Flow)

NEW → ASSIGNED → CALLED → FOLLOW_UP → APPOINTED → ARRIVED → WON
                                                                    ↘ LOST

NEW       = Mới tạo (từ landing page)
ASSIGNED  = Đã phân chi nhánh
CALLED    = Đã gọi (không nghe / đã tư vấn)
FOLLOW_UP = Đang chăm sóc / hẹn gọi lại
APPOINTED = Có lịch hẹn đến
ARRIVED   = Khách đã đến cơ sở
WON       = Có doanh thu
LOST      = Mất khách

⚠️ Lưu ý quan trọng

  • SĐT phải bắt đầu bằng 0 hoặc +84, 10-11 chữ số
  • API trả 400 nếu thiếu field bắt buộc
  • API trả 401 nếu token hết hạn → gọi login lại
  • API trả 403 nếu không đủ quyền (role)
  • API trả 409 nếu SĐT trùng (khi dùng /api/bookings/manual)
  • Tất cả response đều là JSON, mã UTF-8
